# 链表 Linked List

### 为什么链表很重要

- 最简单的动态数据结构
- 更深入的理解引用（或者指针）
- 更深入的理解递归
- 辅助组成其他数据结构



### 链表

数据存储在节点（Node）中

Node拥有两部分，`e`存储真正的数据，`next`指向下一个节点



### 优点

真正的动态，不需要处理固定容量的问题



### 缺点

丧失了随机访问的能力（不能给一个索引就直接访问到）



###数组和链表的对比

#### 数组

数组最好用于索引有语义的情况：scores[2]（快速查询）

最大的优点：支持快速查询



#### 链表

链表不适合用于索引有语义的地方

最大的优点：动态
